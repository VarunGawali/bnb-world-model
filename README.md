# BnB World Model

A GNN-based world model for learned branching in Mixed Integer Programs (MIPs).

The model learns to imitate strong branching using a bipartite Graph Neural Network
trained on Branch-and-Bound trajectories generated by SCIP via Ecole. It achieves
a **264× micro-speedup** over strong branching per node and a **3.7× macro-speedup**
in total solve time on Set Cover instances.

## Architecture

```
B&B Node (bipartite graph)
        │
        ▼
  BipartiteGNN encoder
  (SAGEConv, 3 layers, hidden=128)
        │
   ┌────┴────────────────┐
   ▼                     ▼
h_vars [n_vars, 128]    z [1, 128]
   │                     │
PolicyHead           ValueHead / IntegralityHead / DynamicsGRU
(branching scores)   (dual bound / leaf prob / next latent state)
```

## Setup

```bash
# 1. Clone
git clone https://github.com/<your-username>/bnb-world-model.git
cd bnb-world-model

# 2. Install core dependencies
pip install -e .

# 3. Install Ecole (required for data generation and benchmarking)
conda install -c conda-forge ecole pyscipopt
```

## Quickstart

### Generate Data
```bash
python scripts/generate_data.py --problem set_cover --n 1000
```

### Train
```bash
# Run all four phases sequentially
python scripts/train.py --phase all

# Or run one phase at a time
python scripts/train.py --phase 1   # policy head
python scripts/train.py --phase 2   # value head
python scripts/train.py --phase 4   # joint fine-tuning
```

### Evaluate
```bash
python scripts/evaluate.py --checkpoint checkpoints/phase4_best.pt
```

## Training Phases

| Phase | What trains | Frozen | Key metric |
|-------|-------------|--------|------------|
| 1 | Policy head | — | Top-1 accuracy vs. strong branching |
| 2 | Value head | Encoder + Policy | Spearman ρ on dual bound |
| 3 | Dynamics GRU | Encoder | Latent MSE + cosine loss |
| 4 | All (joint) | — | Total loss, Top-1 accuracy |

## Results (Set Cover, 500×1000, density=0.05)

| Method | Avg Nodes | Avg Time |
|--------|-----------|----------|
| SCIP default | 98.0 | 12.75s |
| Random branching | 1177.5 | 10.64s |
| **BnB-WM (ours)** | **143.5** | **3.45s** |

Micro-speedup vs. strong branching per node: **264×**
